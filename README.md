# 🏏 Cricket ML — IPL Prediction Suite

A production-grade machine-learning pipeline for IPL match analysis, built on
636 matches and 150 000+ ball-by-ball deliveries. Four ML models are served
through a FastAPI REST backend and explored through an interactive Streamlit
dashboard.

---

## Model Performance

| Model | Algorithm | Target | Key Metric |
|---|---|---|---|
| Match Winner | XGBoost (classifier) | Team 1 win (pre-match) | AUC **0.797** |
| Score Predictor | LightGBM (regressor) | Final 1st-innings score | R² 0.52, MAE ≈ 18 runs |
| Win Probability | LightGBM (classifier) | Batting team wins (live) | AUC **0.822** |
| POTM Classifier | XGBoost (classifier) | Player of the Match | AUC **0.972**, PR-AUC 0.67 |
| GRU Score Predictor | PyTorch GRU | Final 1st-innings score (sequence) | MAE ≈ 18 runs, R² 0.33 |

**Match Winner AUC journey:** 0.530 (base, 9 features) → **0.797** (v3, 20 rolling player-form features, 756 matches).

> **Note on GRU vs LightGBM:** The GRU score predictor enables over-by-over inference (predict at any over 1–20) which LightGBM cannot — it's an architectural upgrade. On 756 matches the GRU marginally underperforms the static LightGBM snapshot due to limited sequence count. Adding Cricsheet data (planned — ~5,000+ matches) is expected to push GRU MAE below LightGBM's 17.9 runs.

## Quickstart

### Prerequisites

- Python 3.11+
- The four `.pkl` files in `models/` and six `.parquet` files in
  `data/processed/` must already exist.  
  If they don't, run the notebooks in order first:
  ```
  python notebooks/01_eda.py
  python notebooks/02_feature_engineering.py
  python notebooks/03_modeling.py
  ```

### Option A — Local (two terminals)

```bash
# Install Tier 3 dependencies once
pip install -r requirements_tier3.txt

# Terminal 1 — FastAPI
set PYTHONPATH=.                        # Windows
# export PYTHONPATH=.                   # macOS / Linux
uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload

# Terminal 2 — Streamlit
streamlit run streamlit_app.py
```

| Service | URL |
|---|---|
| FastAPI interactive docs | http://localhost:8000/docs |
| FastAPI health check | http://localhost:8000/health |
| Streamlit dashboard | http://localhost:8501 |

### Option B — Docker Compose (full stack)

```bash
docker compose up --build
```

Both services start automatically. The Streamlit container waits for the API
health check to pass before becoming available.

## API Reference

Base URL: `http://localhost:8000`

All prediction endpoints accept and return JSON.

---

### `GET /health`

Returns API status and which models are loaded.

**Response**
```json
{
  "status": "ok",
  "models_loaded": {
    "match_winner": true,
    "score_predictor": true,
    "win_probability": true,
    "potm_classifier": true
  },
  "teams": ["CSK", "DC", "KKR", "MI", "..."],
  "venues": ["Eden Gardens", "Wankhede Stadium", "..."]
}
```

---

### `GET /matches`

Returns all match IDs available for the win-curve endpoint.

**Response**
```json
{
  "match_ids": [1, 2, 3, "..."],
  "count": 634
}
```

---

### `POST /predict/match-winner`

Pre-match win probability for both teams.

**Request**
```json
{
  "team1": "MI",
  "team2": "CSK",
  "venue": "Wankhede Stadium",
  "toss_winner": "team1",
  "toss_decision": "bat",
  "season": 2019
}
```

`toss_winner` must be `"team1"` or `"team2"` (not the team abbreviation).  
`toss_decision` must be `"bat"` or `"field"`.

**Response**
```json
{
  "team1": "MI",
  "team2": "CSK",
  "team1_win_probability": 54.3,
  "team2_win_probability": 45.7,
  "model_version": "match_winner_v1"
}
```

---

### `POST /predict/score`

Predicted final 1st-innings score, given the state at the end of over 10.

**Request**
```json
{
  "batting_team": "MI",
  "venue": "Wankhede Stadium",
  "season": 2019,
  "runs_10": 68,
  "wickets_10": 2,
  "boundaries_10": 9
}
```

**Response**
```json
{
  "predicted_final_score": 174.5,
  "confidence_interval_low": 161.5,
  "confidence_interval_high": 187.5,
  "current_rr": 6.8,
  "projected_naive": 136.0,
  "model_version": "score_predictor_v1"
}
```

`confidence_interval_*` is a ±13-run MAE-based heuristic interval.

---

### `GET /predict/win-curve/{match_id}`

Over-by-over win probability for the chasing team in a historical match.

**Path parameter:** `match_id` — integer from `GET /matches`

**Response**
```json
{
  "match_id": 335982,
  "batting_team": "MI",
  "bowling_team": "CSK",
  "curve": [
    {"over": 1, "win_probability": 48.2},
    {"over": 2, "win_probability": 51.7},
    "..."
  ],
  "actual_winner": "MI",
  "model_version": "win_probability_v1"
}
```

---

### `POST /predict/potm`

Player of the Match probability for each player in a completed match.

**Request**
```json
{
  "players": [
    {
      "player_name": "Rohit Sharma",
      "runs_scored": 78,
      "balls_faced": 48,
      "wickets_taken": 0,
      "runs_given": 0,
      "balls_bowled": 0,
      "player_won": 1
    },
    {
      "player_name": "Jasprit Bumrah",
      "runs_scored": 4,
      "balls_faced": 3,
      "wickets_taken": 4,
      "runs_given": 18,
      "balls_bowled": 24,
      "player_won": 1
    }
  ]
}
```

**Response**
```json
{
  "predicted_potm": "Jasprit Bumrah",
  "players": [
    {
      "player_name": "Jasprit Bumrah",
      "potm_probability": 81.4,
      "strike_rate": 133.33,
      "economy": 4.5,
      "rank": 1
    },
    {
      "player_name": "Rohit Sharma",
      "potm_probability": 68.2,
      "strike_rate": 162.5,
      "economy": 0.0,
      "rank": 2
    }
  ],
  "model_version": "potm_classifier_v1"
}
```

Players are returned sorted by `potm_probability` descending.

## Encoding Compatibility Note

The trained models use `pd.Series.astype("category").cat.codes` for team and
venue encoding. Codes are assigned in **alphabetically sorted order** of unique
values seen in the training data. `api/main.py` reconstructs these maps at
startup from the same processed parquets, so inference encoding is guaranteed
to match training encoding without storing a separate encoder artefact.
